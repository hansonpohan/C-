四、了解Git與GitHub和key的綁定

 

l   Git是本機上用來做版本控管的軟體，可單獨使用，GitHub是一個線上平台，可與Git做整合，讓您可以在本機與雲端間同步具多個版本的專案資料夾，GitHub不僅有雲端備份的好處，還可在雲端分享您的專案，或透過雲端實作多人開發的應用

l   要在本機Git與雲端GitHub間做同步，就要先綁定雙方的key
過程：在本機Git建立public key與private key，上傳public key至雲端GitHub，以後透過本機的private key與雲端的public key，兩邊就可同步資料

 

1.         製作鑰匙

甲、在本機開啟Git

乙、執行指令
ssh-keygen -t ed25519 -C "pinjuw@gmail.com"
一直按enter
or
ssh-keygen -t rsa -b 4096 -C "pinjuw@gmail.com"

丙、ls ~/.ssh/
id_ed25519  id_ed25519.pub

2.         啟動ssh agent，把private key加入ssh agent

甲、啟動ssh agent
eval `ssh-agent -s`

乙、把private key加入ssh agent
ssh-add ~/.ssh/id_ed25519

3.         上傳public key至雲端GitHub

甲、查看public key
cat ~/.ssh/id_ed25519.pub
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIM9GZrre1UHVCJiz9WnlNvnIQ2YoOPXnVyMWkQFDv4Eo pinjuw@gmail.com

乙、copy public key

丙、在GitHubà點Settings

丁、點左邊「SSH and GPG keys」

戊、點New SSH key

己、貼上public keyà點Add SSH key


4.         測試雲端public key與本機private key是否有成功對應

甲、在本機Git執行
ssh -T git@github.com

乙、輸入yes

丙、….
You've successfully authenticated, but GitHub does not provide shell access.

附錄一：取消或刪除分支中最後幾個版本

 

git revert HEAD：取消某分支最後一個版本（最後一個commit）後，目前會處於分支的最新狀態（main），而且還會保留最後一個版本

1.         在本機Git main分支建立2個新版本來練習

甲、先確定目前在main分支

git add .
git commit -m "add 11"

乙、echo 222 >> 22
git add .
git commit -m "add 22"

丙、ls  ß可以看到11和22兩個檔案

2.         git revert HEAD：取消最後一個版本

甲、git revert HEAD

乙、會跳出vi介面，只是讓我們輸入commit message，直接存檔就可

丙、ls  ß只剩下11，已取消最後一個版本22

丁、查看Sourcetree

說明：刪除22（取消最後一個版本），目前還處於分支的最新狀態（main），萬一後悔，視需要還可git checkout HEAD~，回到add 22的舊版本，找回22的檔案，revert相對比較安全

附錄二：開發流程Git Flow

 

Git Flow開發流程：因為多人開發，每個人的習慣不同，可能建立一大堆分支和版本，有可能已經修好的bug，在下次merge時，又出現相同的bug，Gig Flow給一個標準，讓團隊共同遵循，減少問題產生

分支名稱介紹

l   Develop分支：做為整個專案開發的基礎，不會在上面直接開發功能commit，而是從develop開一個新分支feature來開發新功能，在feature分支開發完新功能並測試沒問題後，才會合併回develop分支

l   Feature分支：每個新功能都會從develop分支開一個feature分支來開發，開發完會合併回develop分支，feature分支常命名為：feature/xxx-function

l   Main分支：或稱master，為預設的分支，存放穩定可發佈上線的版本，不可在上面開發功能commit，只能從develop或release分支合併過去

l   Release分支：develop分支可合併到release分支供上線前最後的測試，測試修bug完，最後還會合併到main（master）與develop分支

l   Hotfix分支：上線後被發現bug問題，會從main開一個hotfix分支來修復問題，修復完再合併回main（master）與develop分支

l   main與develop為常駐分支不會刪除，其他分支只要完成，視需求都可刪除
